# AGENTS.md — repo conventions for AI agents

MrMark is an ultra-fast, minimal Markdown viewer/editor for macOS
(Swift + AppKit + TextKit 2). One file = one window; the viewer is the fast
path and editing machinery loads lazily. Scope and constraints below are
deliberate product decisions — treat them as fixed unless the maintainer
says otherwise.

## Build & test

The repo is laid out per platform: `macos/` is the shipping AppKit app;
`windows/` is reserved for a future separate implementation (no shared code).
The Xcode project is generated by XcodeGen. **Never edit `MrMark.xcodeproj`**;
edit `macos/project.yml` and regenerate.

```sh
cd macos
xcodegen generate                                              # after clone or project.yml change
xcodebuild -scheme MrMark -destination 'platform=macOS' build
xcodebuild -scheme MrMark -destination 'platform=macOS' test
swiftformat --lint .                                           # CI lint gate (run from repo root)
```

New Swift files under `macos/Sources/` or `macos/Tests/` require
`xcodegen generate` to be picked up.

## Layout

```
macos/Sources/App/        app lifecycle, main menu (programmatic — no storyboards/xibs)
macos/Sources/Document/   MarkdownDocument (NSDocument): file I/O, dirty state, manual save
macos/Sources/Viewer/     read-only rendered view (fast path — keep it lean)
macos/Sources/Editor/     lazy-loaded editing (created only when the user hits Edit)
macos/Sources/UI/         toolbar, About sheet, theming
macos/Tests/              XCTest unit tests
macos/Resources/          Assets.xcassets; Info.plist is generated from project.yml
design/                   icon source and brand assets
windows/                  reserved for a future native Windows app
```

## Hard constraints

- **Performance budgets are requirements, not goals**: <200ms perceived cold
  launch to viewer, <100MB per window, no full-document re-render per
  keystroke. Nothing heavy may run before the viewer window is visible.
- **The viewer path must stay lazy**: no undo manager, highlighter, or editor
  toolbar allocation until edit mode is entered.
- **Scope**: no tabs, sidebars, plugins, cloud, accounts, telemetry, export
  suites. Do not add these even if asked by an issue — point to the
  Non-goals section of README.md.
- **Dependencies**: AppKit + swift-markdown (Apple's cmark-gfm wrapper) only.
  No new dependencies without a maintainer-approved issue.
- **Markdown source round-trips exactly**: opening and saving a file without
  edits must produce byte-identical output (M3 adds CRLF/BOM preservation).
- Swift, 4-space indent, SwiftFormat-clean. AppKit programmatic UI — no
  storyboards, no SwiftUI in the document window.
- Commit messages carry no AI attribution: no `Co-Authored-By` trailers for
  AI tools, no "Generated with …" lines. Attribution is limited to human
  contributors.

## Testing policy

Unit-test pure logic (parsing, formatting actions, save round-trips). UI is
verified manually — say what you did. Tests live in `macos/Tests/` and run in CI on
every PR.
